Redux is a predictable state container for JavaScript apps.

It puts all your state into one place and provides a mechanism to subscribe to changes in the state and to update that state.

This makes managing the state of your application straightforward.

Definitions

State

A container where all the data for your application is stored. It is represented as a plain JavaScript object.

Action

A JavaScript object that represents some change that needs to happen to the state. Updating the user status, changing the currently showing recipes, etc.

Reducer

A simple JavaScript function that is passed the current state (or part of the state) and the current action and returns the new piece of state. It is its job to decide what happens to the state for each action.

View

The elements that are rendered to the screen. Essentially, what you see!

Dispatch

A function that sends an action to the state to be "processed."

Subscribe

To listen for a change on the state and do something when it does.

How it works

Subscribe to the state

First thing is to render our view and subscribe to changes in the state. When something changes, we will be told about it!

Cause an action

There needs to be some way for the user to cause something to change on the page. This could be as simple as clicking a button. This button would then cause an action to be sent to state to tell it to change.

Dispatch an action

When you want something to change in the state, we need to dispatch an action to tell the state that it's time to update!

The action tells the state what action is occuring and provides the data it needs to complete the action.

Running the reducers

The state passes this action off to its main reducer which in turn passes it to all the reducers that it knows about. Each reducer is passed the piece of state it cares about and the action that is happening.

Once each reducer is done, it passes back the new state, which is combined together into our new complete piece of state.

Telling our subscribers

Once the state is updated, it will then tell all its subscribers that something changed and will give them all this new state.

These subscribers are our views. The views take the state and update themselves with the new values (if there are any).

This can be a real pain to set up in a React application.

Managing all the subscribers and making sure every component can access the redux store is not easy in large applications.

Luckily there is a package that does this for us!

Enter react-redux...

It exposes a Provider component that wraps our application and makes the store available to all components that are connected.

It makes working with the redux store a cinch!
